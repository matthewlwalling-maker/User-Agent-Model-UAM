# UAM Bridge Skills Rollout Plan

Version: `0.1.0`  
Status: rollout lab candidate  
Evidence ceiling: design-time

## Phase 0: Source Skeleton

- Create `KERNEL.md`, `MANIFEST.yaml`, eight skill contracts, lenses, eval fixtures, adapter placeholders, and rollout docs.
- Do not install, activate, or replace any provider profile.
- Preserve explicit-command pilot posture.

## Phase 1: Strengthen Canonical Skills

- Expand each `SKILL.md` using donor resources and realistic examples.
- Add compact and material output examples.
- Keep donor skills as lenses or patterns, not one-for-one global commands.

## Phase 2: Build Evals Before Quality Claims

- Expand routing, object-integrity, evidence-ceiling, degradation, overactivation, and cross-model parity cases.
- Add blind value-proposition cases that compare the frozen bridge package against a same-model baseline without revealing output identity to the evaluator.
- Run design-time validation over the package.
- Forward-test on realistic prompts.

## Phase 3: Diagnose and Patch Failures

Place each failure at the correct layer:

- Kernel/router.
- Skill procedure.
- Lens.
- Resource.
- Adapter.
- Eval.
- Governance.

## Phase 4: Codex Adapter First

- Create Codex adapter from the canonical manifest and skill files.
- Keep explicit command invocation first.
- Do not copy into global skill locations without explicit authorization.

## Phase 5: Other Provider Adapters

- Draft Claude Code, ChatGPT, Gemini, and generic adapters from the same canonical source.
- Each adapter must declare a dated capability profile.
- Degrade honestly when file writing, retrieval, code execution, memory, or isolation is unavailable.

## Phase 6: Explicit-Command Pilot

- Use the eight commands directly.
- Record confusion, overactivation, boundary drift, and evidence overclaims.
- Keep broad implicit routing disabled until eval gates pass.

## Phase 7: v1 Gate

- Run the v1 release gate.
- Require blind comparative evidence before claiming the skillsets improve behavioral utility beyond baseline.
- Preserve rollback instructions.
- Record limitations and known non-goals.
